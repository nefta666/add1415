Instalación LDAP

Configuramos el equipo en el que usaremos LDAP con IP estática.


Instalamos LDAP y las utilidades de LDAP desde apt-get. El paquete "ldap.4.2 util", no se encontraba disponible en apt-get y fue necesario instalar una versión posterior mediante Synaptic. Luego Usamos el comando "dpkg-reconfigure slapd" para entrar en el asistente de configuración de LDAP


Durante la instalación de LDAP le diremos que no queremos omitir la configuración del servidor.


El siguiente paso será introducir el nombre de dominio que se usará para LDAP


Introduciremos una contraseña de administrador en el siguiente paso.


En caso de que nos pregunte si queremos mover la base de datos antigua, le diremos que si.


Luego, el asistente nos preguntará si deseamos activar el protocolo LDAPv2. Marcaremos que no, ya que está obsoleto y no lo vamos a usar.


El último paso del asistente, nos preguntará que hacer con la base de datos en caso de que se elimine el paquete slapd. En nuestro caso, hemos marcado la opción de no borrar.


Ahora que ya se ha instalado LDAP y el resto de utilidades, vamos a comprobar si se para y arranca con normalidad.


Para finalizar la instalación, instalamos los paquetes de Apache y PHP que no habíamos instalado previamente.


Introducción Datos en LDAP

Comprobamos que tenemos acceso al directorio de LDAP introduciendo en el navegador la dirección "localhost/phpldapadmin"


Para cambiar el nombre del directorio, que aparece como "dc=example dc=com" modificaremos el archivo "/etc/phpldapadmin/config.php". En la linea 300 del archivo cambiamos los valores por nuestro dominio.


Lo siguiente será introducir datos en LDAP, para ello descargaremos JXplorer desde Synaptic.



Una vez dentro de JXplorer, dentro de nuestro directorio, crearemos las unidades organizativas "usuarios" y "grupos".


Dentro de groups, crearemos otras dos unidades organizativas llamadas, "alumnos" y "profesores"


Dentro de grupos, crearemos otras dos unidades organizativas llamadas, "alumnos" y "profesores"


Por último, crearemos varios usuarios dentro de la unidad organizativa "users"
