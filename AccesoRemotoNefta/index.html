RDP

De Windows a Windows

Lo primero que haremos será configurar las ip's ambos equipos



Comprobamos que efectivamente hay conexión incluso
podemos hacer ping entre las dos maquinas


Habilitamos las conexiones entrantes a escritorio remoto en la maquina que hara de servidor
y desde la maquina que hara de cliente abrimos Escritorio Remoto.
Introducimos la IP del servidor y pulsamos en conectar



Debería aparecer la siguiente pantalla, ponemos el nombre de usuario del servidor y pulsamos en Aceptar.


Cuando conecte veremos el equipo del servidor y se cerrara la sesion real del equipo remoto.


De Linux a Windows

Lo primero que haremos será habilitar las conexiones entrantes para Escritorio Remoto


Luego configuraremos en Windows la IP de forma estática


Hacemos lo mismo en Ubuntu


Añadimos al usuario Administrador al grupo de usuario de Acceso Remoto
(Aunque debería venir añadido por defecto)


Abrimos Remmina en Ubuntu y colocamos la IP de la maquina a la que nos vamos a conectar
y el nombre de usuario y contraseña.


Se abrira la sesión de ese usuario y se cerrara en la maquina remota.


De Windows a Linux

Ya que las conexiones IP estan configuradas de pasos anteriores, lo primero que tendremos
que hacer sera instalar el servidor XRDP en Ubuntu para que desde Windows se pueda acceder a el.


Una vez el servidor XRDP esté instalado y puesto en marcha, desde Windows abriremos Escritorio remoto
y pondremos la IP de la maquina Ubuntu. En la ventana que aparecera seleccionaremos vnc=any en la pestaña de modulo
y colocaremos usuario y contraseña. Tras varios intentos y probando varios modulos no se consiguió conectar.


VNC

De Windows a Windows

Para empezar, primero deberemos instalar VNC (servidor y cliente al menos en una maquina), en ambas maquinas.
Ya que estamos usando las mismas maquinas que en casos anteriores, la configuración IP es la misma.
Una vez tengamos instalado VNC en ambas maquinas, ejecutamos el servidor en una de ellas y en la otra el cliente.
En la ventana que aparece a continuacíon colocamos la IP de la maquina que hace de servidor y pulsamos en Connect.


Si no da ningun error deberiamos ver la pantalla de la maquina servidor.


De Linux a Linux

Descargamos e instalamos VNC en ambas maquinas Linux.
Configuramos IP estática para la nueva maquina que vamos a usar.
Ejecutamos el servidor VNC y cambiamos a la otra máquina virtual.


Ejecutamos el VNC Viewer, colocamos la IP de la máquina que ejerce de servidor y pulsamos en Connect.


Servidor de Terminales

En Windows

Lo primero que haremos será dirigirnos a Administrar el Servidor, y desde ahí a Agregar funciones.
Desde ahí seleccionamos Terminal Services y continuamos.


En la siguiente pantalla, marcamos las dos primeras opciones y continuamos con la instalación.


Como no vamos a requerir seguridad para la práctica, marcamos la opción no requerir autenticación a nivel de red.


Nos saltaremos la pantalla de la licencia y marcamos Configurar mas adelante.


Como no trabajamos con dominios, esta pantalla la dejaremos como esta y finalizamos la instalación.


Una vez finalizada la instalación de Terminal Services, crearemos dos usuarios locales y los añadiremos
al grupo Usuarios de escritorio remoto.


Lo siguiente lo haremos ya desde Windows 7. Abriremos Escritorio remoto y colocaremos la IP de la máquina de Windows 2008.
Al darle a Conectar debería aparecer la siguiente pantalla.


Pulsamos en Si y se no abrirá la pantalla de inicio de sesión de usuario. Iniciaremos sesión con uno de los
que acabamos de crear en Windows 2008.


En Linux

Desde Linux, como ya tenemos del caso previo todo configurado, solo tendremos que abrir Remmani
y colocar la IP de la máquina Windows 2008, nombre de usuario y contraseña.


Pulsamos en Conectar y debería aparecer la pantalla de la máquina windows 2008.


RemoteAPP

Para configurar RemoteAPP, primero nos dirigiremos a Herramientas administrativas/Terminal Services y desde ahí
a Administrador de RemoteAPP de TS. Nos dirigimos a configuración de RemoteAPP y colocamos el en la siguiente pantalla colocamos el nombre del servidor.


Ahora en la pantalla principal, pulsaremos sobre Agregar programas RemoteAPP, se abrirá un nuevo asistente.
En el prodremos seleccionar el programa o programas que queremos ejecutar de forma remota desde la maquina cliente. Seleccionamos el que queramos y pulsamos sobre siguiente.


Revisamos que la configuración sea la correcta y finalizamos.


Nuevamente en la pantalla principal de RemoteAPP, veremos el programa que hemos seleccionado.
Haremos click derecho sobre el y pulsaremos sobre Crear archivo .rdp y se abrirá otro nuevo asistente.



En este nuevo asistente podremos configurar el paquete de RemoteAPP.
Si requerirá autenticación del servidor para ejecutarse, si se desea colocar un certificado al paquete para firmarlo o la puerta de enlace.


En este caso,configuraremos la IP del servidor, tambien desactivaremos la autenticación del servidor.


Una vez hechos los cambios, pulsaremos en siguiente y luego en finalizar para terminar la creación del paquete.


Pasamos el paquete "Calculadora(1).rdp" a la maquina cliente y lo ejecutamos. Nos aparecerá el siguiente mensaje avisandonos de que el paquete podría
resultar dañino, ya que al no estar firmado no se puede identificar a su editor. Pulsamos en conectar.


Se abrirá el programa que hayamos seleccionado remotamente desde la maquina del servidor.
